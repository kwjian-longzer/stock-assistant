# GLM-5.2 调研报告

> 调研日期: 2026-06-25  
> 调研目的: 评估 GLM-5.2 是否适合用于股票分析项目的报告撰写

---

## 一、模型概览

GLM-5.2 是智谱AI于 2026年6月13日 正式全量开放的最新一代旗舰大模型 [1]。

| 维度 | 规格 |
|------|------|
| 架构 | MoE（混合专家），744B 总参数，40B 激活参数 |
| 上下文窗口 | 100万 Token（真正可用，非标称值） |
| 开源协议 | MIT（可商用、可修改、可本地部署） |
| 训练数据 | 28.5万亿 Token，华为昇腾910B国产算力 |
| 发布日期 | 2026-06-13 |

## 二、核心能力

### 基准测试成绩 [1]

| 测试项目 | GLM-5.2 | 对比 |
|---------|---------|------|
| SWE-bench Verified | 77.8% | 开源模型第一，接近 Claude Opus 4.7(80.9%) |
| Humanity's Last Exam (带工具) | 50.4% | 全球第一 |
| Browp (浏览器Agent) | 75.9% | 全球第一，Claude 67.8% |
| Intelligence Index v4.0 | 50分 | 首个突破50分的开源模型 |

### 核心优势

1. **1M上下文真正可用**：在95万Token位置插入随机字符串仍能准确召回，行业唯一 [1]
2. **中文能力最强**：国产模型对中文理解天然优势，超过GPT-4o和Claude [1]
3. **Agent能力全球第一**：Browp测试75.9%，适合多步骤自动化工作流 [1]
4. **代码能力开源第一**：SWE-bench 77.8%，能理解整个代码库架构 [1]

### 相对短板

1. 多模态能力弱于GPT-4o（暂不支持图像分析）[1]
2. 深度推理略弱于Claude Opus（差距很小）[1]

## 五、对股票分析项目的适用性评估

### 高价值场景

1. **周报撰写**：1M上下文可一次性加载本周所有报告(7篇×6000字=4.2万字≈8万Token) + 电报归档(数千条) + 数据摘要 + analysis_prompt.md，无需截断
2. **VIP研报分析**：VIP文章全文+催化关键词+Tushare全量股票基础信息(5000+只)可一次性加载
3. **电报归档分析**：当日全天电报(可能数百条)可完整加载分析
4. **跨日趋势分析**：可同时加载多日数据进行趋势对比

### 中等价值场景

1. **日常报告撰写**：当前data_summary.json约2-5万字符(4-10万Token)，DeepSeek的128K窗口已够用
2. **推理链质量**：DeepSeek的深度推理略优，但GLM-5.2的Agent能力可弥补

### 不适用场景

1. 需要图像分析的图表解读（GLM多模态弱）

## 六、建议

### 短期建议（v2.0不切换）
- 当前 DeepSeek-V4-Pro 在日常报告中表现良好，128K上下文足够
- 推理链质量优化（任务5）已通过 analysis_prompt.md 规范化解决
- 切换模型有适配风险，v2.0不建议

### 中期建议（v2.1评估切换）
1. **周报专用**：考虑用 GLM-5.2 撰写周报，利用其1M上下文加载全周数据
2. **VIP深度分析**：用 GLM-5.2 分析VIP研报全文，利用其长文本+中文优势
3. **API成本评估**：月均报告约90篇（3篇/天×30天），每篇消耗约5万Token，月成本约:
   - 输入: 90×5万 = 450万Token × ¥8/百万 = ¥36
   - 输出: 90×1万 = 90万Token × ¥28/百万 = ¥25.2
   - 月总成本约 ¥61.2（不含峰值系数）

### 长期建议（v3.0）
- 如果 GLM-5.2 后续增加多模态能力，可全面切换
- MIT开源协议支持本地部署，未来可考虑私有化部署以降低成本和数据安全
